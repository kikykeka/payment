# Purchase History: Current & Future Solution

## Текущее Решение (Temporary - Работает Сейчас)

### Что Сделано

История покупок теперь хранится в **localStorage, привязанном к адресу кошелька**, а не к домену.

**Ключ хранения**: `solestate_purchases_{walletAddress}`

### Как Это Работает

1. **При подключении кошелька** - загружается история для этого конкретного адреса
2. **При покупке токенов** - запись сохраняется под ключом текущего кошелька  
3. **При отключении** - история остается в браузере для следующего подключения

### Преимущества

✅ История сохраняется для каждого кошелька отдельно
✅ Работает на текущем смарт-контракте без изменений
✅ Быстрый доступ к истории (нет запросов к блокчейну)
✅ Можно использовать разные кошельки в одном браузере

### Ограничения

❌ История привязана к браузеру (не синхронизируется между устройствами)
❌ Может быть удалена пользователем (очистка кеша)
❌ Не работает в режиме инкогнито
❌ Не является "source of truth" - блокчейн остается главным

## Долгосрочное Решение (Recommended - Требует Обновления Контракта)

### Концепция

Хранить **всю историю покупок on-chain** в Solana blockchain.

### Два Подхода

#### Подход 1: Account-Based (Рекомендуется)

Создать новый тип аккаунта `PurchaseRecord` в смарт-контракте:

**Преимущества:**
- Постоянное хранение
- Легко читать через program.account API
- Полный контроль над данными

**Недостатки:**
- Требует rent (~0.002 SOL на запись, возвращается при закрытии)
- Увеличивает сложность контракта

**Структура:**
```rust
#[account]
pub struct PurchaseRecord {
    pub investor: Pubkey,
    pub property: Pubkey,
    pub token_mint: Pubkey,
    pub tokens_purchased: u64,
    pub price_per_token_lamports: u64,
    pub total_paid_lamports: u64,
    pub timestamp: i64,
    pub bump: u8,
}
```

**PDA Seeds:** `["purchase", investor, property, timestamp]`

#### Подход 2: Event-Based (Дешевле)

Эмитировать события при каждой покупке:

**Преимущества:**
- Нет rent cost
- Минимальные изменения в контракте
- Более дешевые транзакции

**Недостатки:**
- Требует индексер (Helius, SimpleHash, или собственный)
- События могут быть сложнее запросить
- Зависимость от внешнего сервиса

**Структура:**
```rust
#[event]
pub struct TokensPurchased {
    pub investor: Pubkey,
    pub property: Pubkey,
    pub token_mint: Pubkey,
    pub tokens: u64,
    pub total_cost: u64,
    pub timestamp: i64,
}
```

### Необходимые Изменения

См. `BLOCKCHAIN_HISTORY_UPGRADE.md` для полной инструкции по обновлению смарт-контракта.

## Миграция: Текущее → Блокчейн

### Шаг 1: Обновить Смарт-Контракт
- Добавить PurchaseRecord account или события
- Обновить purchaseTokens для записи истории
- Деплой на devnet для тестирования

### Шаг 2: Обновить IDL
Заменить IDL в `lib/wallet-context.tsx` новой версией с PurchaseRecord

### Шаг 3: Создать Функцию Чтения
```typescript
export async function fetchPurchaseHistoryFromBlockchain(
  wallet: any
): Promise<PurchaseRecord[]> {
  const program = new Program(IDL, provider)
  const investor = wallet.publicKey
  
  const purchases = await program.account.purchaseRecord.all([{
    memcmp: {
      offset: 8,
      bytes: investor.toBase58(),
    },
  }])
  
  return purchases.map(formatPurchaseRecord)
}
```

### Шаг 4: Обновить Wallet Context
Заменить `loadPurchases` вызовом `fetchPurchaseHistoryFromBlockchain`

### Шаг 5: (Опционально) Миграция Данных
Создать admin функцию для миграции существующих localStorage записей в блокчейн

## FAQ

**Q: Почему история исчезает при переходе на другой домен?**  
A: Теперь не должна! История привязана к адресу кошелька, а не к домену. Убедитесь что подключаете тот же кошелек.

**Q: Что если я очищу кеш браузера?**  
A: История из localStorage удалится. Поэтому важно перейти на блокчейн-хранение для production.

**Q: Могу ли я использовать несколько кошельков?**  
A: Да! Каждый кошелек имеет свою отдельную историю.

**Q: Синхронизируется ли между устройствами?**  
A: Нет в текущем решении. Для синхронизации нужно обновить контракт.

**Q: Что произойдет с историей после обновления контракта?**  
A: Старая история останется в localStorage. Новые покупки будут в блокчейне. Можно создать миграцию.
